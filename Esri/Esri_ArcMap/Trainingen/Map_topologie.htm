Map topologie

Over het maken van map topology

Map topology maakt topologische relaties aan tussen delen van objecten die samenvallen (coincident = dezelfde ruimte delen). Je kunt de objectklassen opgeven die mee moeten doen. Je kunt ook een afstand opgeven, clustertolerantie genaamd, die aangeeft hoe dicht punten bij elkaar moeten liggen om op elkaar gelegd te worden. Als je map topology aanmaakt wordt de clustertolerantie gebruikt om te bepalen welke delen van de objecten samenvallen en welke segmenten (edges) en knooppunten (nodes) worden gedeeld.

Er zijn twee stappen voor het maken van map topology:

  1. Geef aan welke feature classes (kaartlagen) mee moeten doen in de topology.
  2. Specificeer een cluster tolerantie. De cluster tolerantie is de afstand waarbinnen objecten samenvallen. In de geodatabase topology is een waarde van 15 cm ingesteld en is dus ook een goede waarde voor de map topology.

 

Als je map topology hebt gedefinieerd is dat zichtbaar in de Topology lijst als <Map Layers>. Je kunt maar 1 map topology definiëren tijdens een edit sessie.

 

Map topology versus geodatabase topology

Een geodatabase topology bevat een set van regels die de relatie beschrijven tussen objecten in een objectklasse. Geodatabase topology wordt gemaakt in ArcCatalog en kan in ArcMap als een kaartlaag (layer) worden toegevoegd, net als andere data. Nadat je bewerkingen op objecten hebt uitgevoerd valideer je de database topologie om overtredingen van de regels op te sporen. Fouten kun je oplossen of als uitzondering markeren.

 

Map topology is tijdelijk en bestaat alleen tijdens de duur van een edit sessie. In tegenstelling tot geodatabase topology wordt map topology niet permanent in de database opgeslagen of weergegeven als een kaartlaag. Map topology kent geen regels en kent geen validatieproces. De tools voor valideren en oplossen van geodatabase topology zijn uitgegrijsd als je met map topology werkt.

 

Wanneer gebruik je map topology

Map topology is handig als je topologische veranderingen wilt uitvoeren in situaties waarin je geen database topology kunt toepassen.

 

Werken met map topology

Nadat je map topology hebt aangemaakt kun je de Topology Edit tool gebruiken om gedeelde edges en nodes van objecten te wijzigen. Wijzigen van een edge of node die door meerdere objecten wordt gedeeld geeft als resultaat meerdere gewijzigde objecten. Je geeft geen regels op voor map topology. Alle edges of punten van objecten in de map topology die binnen de clustertolerantie vallen worden beschouwd als zijnde topologisch gedeeld. Omdat er geen topologieregels zijn is er geen noodzaak om map topology te valideren en er worden dus ook geen fouten gevonden.

Op geometrieniveau gaat topologie over simpele relaties, zoals samenvallen, bedekken en snijden, tussen geometrieën (punten, lijnen en vlakken). Terwijl alle eenvoudige geometrieën meedoen, wordt tijdens topologie wijzigingen alleen gereageerd op:

  • Edges—lijnsegmenten die lijnen of vlakken definiëren.
  • Nodes—eindpunten van lijnsegmenten.
  • Pseudonodes—een knooppunt die 2 lijnsegmenten verbindt of een logische opdeling (logical split) gedefinieerd in de topologie buffer (topology cache) tijdens bewerken. Pseudonodes van ondergeschikte soort worden tussenpunten na bewerken.

 

Map topology aanmaken

  1. Start een edit sessie.
  2. Klik de Map topology knop in de Topology toolbar.
  3. Vink de objectklassen aan die mee moeten doen in de map topology.
  4. Alle objectklassen in de actieve edit sessie, behalve annotatie en maatvoering, kunnen meedoen en zijn zichtbaar in het dialoogvenster.

  5. Optioneel, stel een cluster tolerantie in van 0.15 mtr.

    De standaard cluster tolerantie is de kleinst mogelijke cluster tolerance. Het vergroten van de cluster tolerantie kan leiden tot het samenvoegen van meer objecten die als samenvallend worden beschouwd, maar kan onnauwkeurigheden in de hand werken.

  6. Klik OK.
  7. Klik de Topology Edit tool en selecteer de gezamenlijke geometrie van de objecten die je wil bewerken. De map topology wordt aangemaakt voor de objecten die zichtbaar zijn in de extend.
