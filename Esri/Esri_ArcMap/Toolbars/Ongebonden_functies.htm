UpdateAttributes

Met deze tool werk je de attribuutwaarden bij van objecten uit de objectklassen hartlijnen, hartpunten, kruisingsvlakken en wegpunten.

  • Hartlijnen behoren dezelfde attribuutwaarden te hebben als de bijbehorende wegvlakken en weglijnen.
  • Hartpunten behoren alle attribuutwaarden te hebben van de daarop eindigende hartlijnen.
  • Kruisingsvlakken en wegpunten behoren dezelfde attribuutwaarden te hebben als hartpunten.

Deze tool kan zorgen voor veel mutaties. Gebruik 'm alleen als je weet waar je mee bezig bent.

 

Precondities:

Attribuutvalidatie moet zonder fouten zijn afgerond

 

De tool werkt attributen als volgt bij:

  1. werk attributen bij van hartlijnen a.h.v. wegvlakken en weglijnen
  2. werk attributen bij van hartpunten a.h.v. hartlijnen en kruisingsvlakken (alleen hoogteniveau)
  3. werk attributen bij van kruisingsvlakken a.h.v. hartpunten
  4. werk attributen bij van wegpunten a.h.v. hartpunten

 

Je kunt aangeven of je attribuutwaarden wil bijwerken in de AOI (area of interest) of extent.

Voor beide mogelijkheden is een knop aanwezig die je zelf aan de interface kunt toevoegen in Toolbars en contextmenu's aanpassen.

  • Objecten die geen of een foutieve relatie hebben naar een ander object worden niet bijgewerkt.

    Bijvoorbeeld de hartlijnen die twee kruisingsvlakken verbinden.

  • Er worden geen attribuutwaarden overgenomen van objecten die geen of een foutieve relatie hebben naar een ander object.

    Bijvoorbeeld een hartpunt waarop een hartlijn eindigt, maar de hartlijn heeft geen relatie naar een wegvlak.

Attributen van hartlijnen bijwerken

Alleen de attributen van hartlijnen die een relatie hebben met een wegvlak of weglijn worden bijgewerkt. Bijwerken betekent de attribuutwaarden één op één overnemen van het gerelateerde wegvlak of weglijn. Betrokken relatietabellen zijn EDT_REL_WEG_VLAK_HARTLIJN en EDT_REL_WEG_LIJN_HARTLIJN.

Als de gebruiker eventuele fouten in de relaties tussen weg-features nog niet heeft opgelost vóór het updaten van de attributen, kunnen er hartlijnen, wegvlakken en weglijnen zijn met foutieve relaties met andere weg-features. Hieronder vallen ook de hartlijnen zonder relatie. Als de hartlijn die geupdate moet worden of de weglijn of het wegvlak waarvan het de attributen over moet nemen invalide relaties voor de weg-features hebben, wordt de hartlijn niet geupdate. Het aantal hartlijnen waarvoor dit het geval is wordt d.m.v. een rapportagevenster (zie 3.6) aan de gebruiker getoond aan het eind van de attribuut update procedure.Dit aantal omvat ook de features die door een andere oorzaak niet bijgewerkt konden worden. De foutbeschrijving wordt, samen met het featureOID, naar de logfile geschreven.

De volgende attributen worden bijgewerkt:

TYPE_WEG, HOOFDVERKEERSGEBRUIK, FYSIEK_VOORKOMEN, VERHARDINGSBREEDTEKLASSE, VERHARDINGSBREEDTE, GESCHEIDEN_RIJBAAN, ,VERHARDINGSTYPE, AANTAL_RIJSTROKEN, STATUS, NEDERLANDSE_STRAATNAAM, FRIESE_STRAATNAAM, WEGNUMMER_A_WEG, WEGNUMMER_N_WEG, WEGNUMMER_E_WEG, WEGNUMMER_S_WEG, AFRITNUMMER, AFRITNAAM, KNOOPPUNTNAAM, BRUGNAAM, TUNNELNAAM, HOOGTENIVEAU

 

Procedure:

  • Verzamel alle features van de featureclass hartlijnen binnen de extent/AOI
  • Loop door de gevonden hartlijnen, per hartlijn:
    • Heeft de hartlijn valide relaties met andere wegfeatures?

      Nee: aantal hartlijnen dat niet geupdate is +1

      stop

    • Relatie met een weglijn?

      Heeft de weglijn valide relaties met andere wegfeatures?

      Nee: aantal hartlijnen dat niet geupdate is +1

      stop

      Ja: Neem de relevante attributen over

      stop

    • Relatie met Wegvlak?

      Heeft de wegvlak valide relaties met andere wegfeatures?

      Nee: aantal hartlijnen dat niet geupdate is +1

      stop

      Ja: Neem de relevante attributen over

      Stop

    • Geen relatie (hartlijn zonder relatie is niet valide, maar voor de volledigheid toch toevoegen)

      aantal hartlijnen dat niet geupdate is +1

      Stop

    • Bij een exception: aantal hartlijnen dat niet geupdate is +1

 

Attributen van hartpunten bijwerken

Attribuutwaarden worden overgenomen van alle hartlijnen waarvan het start- of eindpunt samenvalt met het hartpunt en die geldige relaties hebben. Samenvallende hartlijnen die geen valide relaties hebben zijn in de eerste stap namelijk niet geupdate, daarom worden de attributen daarvan hier niet gebruikt.

Bij attributen waarvan de multipliciteit veelvoudig is worden de verschillende unieke waarden achter elkaar geplaatst en gescheiden door een ‘|’-teken, bijvoorbeeld: TYPE_WEG=’| hoofdweg|lokale weg| overig|’

Het gaat hierbij om de volgende attributen (allen van het type String):

TYPE_WEG, HOOFDVERKEERSGEBRUIK, FYSIEK_VOORKOMEN, NEDERLANDSE_STRAATNAAM, FRIESE_STRAATNAAM, WEGNUMMER_A_WEG, WEGNUMMER_N_WEG, WEGNUMMER_E_WEG, WEGNUMMER_S_WEG, AFRITNAAM, KNOOPPUNTNAAM, BRUGNAAM, TUNNELNAAM

 

Voor de volgende attributen waarvan de multipliciteit enkelvoudig is wordt een keuze gemaakt uit de over te nemen waarden. De waarde met de hoogste prioriteit wordt overgenomen. Voor de volgende attributen gelden de volgende prioriteiten:

 

VERHARDINGSBREEDTEKLASSE (integer)

  1. 7 meter code = 33
  2. 4 – 7 meter code = 34
  3. 2 – 4 meter code = 35
  4. < 2 meter code = 36

GESCHEIDEN_RIJBAAN (integer)

  1. Ja code = 1006
  2. Nee code = 1007

VERHARDINGSTYPE (integer)

  1. verhard code = 37
  2. half verhard code = 38
  3. onverhard code = 39
  4. onbekend code = 2303

STATUS (integer

  • in gebruik code = 40
  • realisatie: in uitvoering code = 41
  • realisatie: nog niet in uitvoering code = 42
  • buiten gebruik code = 43
  1. onbekend code = 44

 

Voor de volgende attributen waarvan de multipliciteit ook enkelvoudig is wordt de hoogste waarde overgenomen:

VERHARDINGSBREEDTE (double) en AANTAL_RIJSTROKEN (integer).

Voor het attribuut AFRITNUMMER (string) waarvan de multipliciteit ook enkelvoudig is, wordt de waarde van één van de hartlijnen overgenomen. Als geen van de hartlijnen een afritnummer heeft wordt er een lege waarde voor het attribuut ingevuld.

 

Voor het attribuut HOOGTENIVEAU (integer), waarvan de multipliciteit ook enkelvoudig is, geldt een afwijkende werkwijze. Indien er een relatie bestaat naar een kruisingsvlak of wegpunt wordt de waarde van het gerelateerde object overgenomen. Niet alle hartpunten hoeven een relatie met een wegpunt of kruisingsvlak te hebben. Als er geen relatie bestaat wordt de waarde overgenomen van één van de hartlijnen.

Betrokken relatietabellen zijn EDT_REL_WEG_VLAK_HARTPUNT en EDT_REL_WEG_PUNT_HARTPUNT.

Als de gebruiker eventuele fouten in de relaties tussen weg-features nog niet heeft opgelost vóór het updaten van de attributen, kunnen er hartpunten, wegvlakken en wegpunten zijn met foutieve relaties met andere weg-features. Als het hartpunt dat geupdate moet worden of het wegvlak of het wegpunt waarvan het de attributen over moet nemen invalide relaties voor de weg-features heeft wordt er geen hoogte van het gerelateerde feature overgenomen en wordt de waarde overgenomen van één van de hartlijnen. Het aantal hartpunten waarvoor dit het geval is wordt d.m.v. een rapportagevenster (zie 3.6) aan de gebruiker getoond aan het eind van de attribuut update procedure.

Hieronder vallen ook de kruisingsvlakken en wegpunten zonder relatie.         Nee: aantal kruisingsvlakken dat niet geupdate is +1

        stop

        Ja: Neem de attribuutwaarde van het hartpunt over

        Stop

      • Geen relatie? (kruisingsvlak zonder relatie is niet valide, maar voor de volledigheid toch toevoegen)

        aantal kruisingsvlakken dat niet geupdate is +1

        Stop

    • Bij een exception: aantal kruisingsvlakken dat niet geupdate is +1
  • Verzamel alle features van de featureclass wegpunten binnen de extent/AOI
  • Update wegpunten:

Als je de gereedmeldingen wilt verwijderen doe je dat door een kader te ‘trekken’ over de cellen met de knop Voortgang gereed.

Feature Inspector

Op basis van de specifieke Feature Inspector voor LV Beeldmateriaal is de BRT Feature Inspector ontwikkeld. Deze is zelfs zo generiek dat hij niet alleen op BRT data werkt:

alle type objectklassen in ArcMap kunnen gebruikt worden als input om doorheen te navigeren.

 

Gebruik van de BRT Feature Inspector

De BRT Feature Inspector werkt op elke objectklasse in ArcMap, waaronder uiteraard de objectklassen van de BRT.

1. Als eerste stap moet er een selectie worden gemaakt. Deze selectie kan worden gemaakt met de standaard esri functionaliteiten,

bijvoorbeeld door selectie van objecten in een attribuuttabel.

2. Klik vervolgens op het tandwielsymbool (Configureer de feature inspector) in de BRT Feature Inspector toolbar

3. Een configureerscherm wordt getoond om de Feature Inspector in te stellen met de volgende opties en mogelijkheden:

Selectielaag: geeft aan welke selectie ingeladen moet worden in de Feature Inspector. Standaard worden alle geslecteerde objecten ingeladen, maar je kan ook de selectie van één specifieke laag instellen. Het is dus mogelijk om selecties uit meerdere lagen van verschillende geometrietypen te combineren!
Wachttijd (sec): Stelt de doorlooptijd in van de playfunctionaliteit
Zoomen of Pannen Geeft aan of er naar het geselecteerde object wordt toegesprongen in het huidige zoomlevel (pannen) of dat er een nieuw zoomlevel wordt ingesteld
Zoom Factor Voor het zoomen naar vlakken en lijnen. Dit zoomt naar de extent van het object (Zoom Factor = 1) of een aangepaste zoom factor-waarde.
Puntschaal Het zoomlevel waarop geselcteerde punten worden getoond (punten hebben geen extent).

 

Handmatig navigeren met de Feature Inspector:

vorige en volgende feature met de pijl knoppen

 

Snel naar een eerste of laatste feature met de dropdown button

 

Als gevolg van het navigeren centreert de data view zich op het geselecteerde object volgens de ingestelde waarden.

 

Automatisch navigeren met de Feature Inspector:

Driehoek knop start het automatisch navigeren

Blokjesknop stopt het automatisch navigeren

 

Als gevolg hiervan centreert de data view zich op het geselecteerde object volgens de ingestelde waarden. Na het als wachttijd ingestelde aantal seconden wordt automatisch naar het volgende geselecteerde object gecentreerd.
